## ETF操盘看板项目搭建完成

从零搭建了完整的前后端分离ETF看板系统，路径：`c:/Users/smipl/WorkBuddy/etf trading strategy/`

### 后端（Python FastAPI）
- **数据获取**：使用 akshare `fund_etf_hist_sina` 接口获取ETF历史行情，绕过了东方财富接口的网络限制
- **跟踪标的**：40只ETF（9只宽基 + 31只行业），代码池配置在 `backend/app/config.py`
- **指标计算**：
  - 当前涨跌幅
  - 上两个交易日累计涨幅 = (T-1收盘价 - T-2收盘价) / T-2收盘价 × 100%
  - 成交量放大 = 当日成交量 / (前两交易日成交量均值) - 1，再转百分比
  - 综合得分 = 涨跌幅×0.2 + 两日累计涨幅×0.3 + 成交量放大×0.5
- **LLM推荐**：支持通过环境变量配置OpenAI兼容API进行热点分析推荐；未配置时自动回退到基于行情数据的规则推荐
- **API端点**：`POST /api/refresh`（强制刷新）、`GET /api/dashboard`（读取缓存或刷新）

### 前端（React + Vite + Tailwind）

- **表格功能**：支持按宽基/行业筛选，表头点击排序（涨跌幅、两日累计、成交量放大、综合得分）

- **开发服务器代理**：前端 `/api` 自动代理到后端 `127.0.0.1:8000`

### 盘中/盘后数据源区分（2026-04-21 更新）

**问题**：原版用新浪历史日线接口（`fund_etf_hist_sina`），盘中也是用昨天收盘价，无法区分盘中实时和盘后数据。

**解决方案**：
- **交易时段**：改用 `stock_zh_a_minute(symbol='sh/sz+code', period='1')` 获取今日1分钟K线，取最新一根K线收盘价=当前实时价，累计成交量=今日总成交量，昨收通过新浪历史接口单独查
- **非交易时段**：继续用 `fund_etf_hist_sina` 取昨日收盘历史数据
- data_time 前端展示：仅显示日期，后端返回 `data_type: 'realtime' | 'closed'`，前端渲染"收盘"或"盘中(实时)"标签

**WEIGHTS键名踩坑**：`config.py` 定义为 `current_change / two_day_change / volume_expand`（无 pct 后缀），calculator 引用时若写成 `WEIGHTS["current_change_pct"]` 会 KeyError，需严格对应

**volume 字符串拼接 bug**：`stock_zh_a_minute` 的 volume 列是字符串类型，`df["volume"].sum()` 导致字符串拼接，volume 变成 10^3142 量级溢出。修复：`pd.to_numeric(today_df["volume"], errors="coerce").sum()`

**分钟线数据判断扩展**：原 `is_trading_time()` 仅覆盖 9:30-15:00，盘后（15:00-24:00）错误回退到昨日数据。新增 `should_use_realtime_mode()` 覆盖 9:30-24:00，盘后返回 `data_type="closed_today"`，前端显示"今日收盘"标签
### 分钟线数据升序/降序 bug（2026-04-21 修复）

**问题**：`ak.stock_zh_a_minute` 返回的分钟线数据是**升序排列**（从早到晚），但 `fetch_realtime_minute` 错误地假设了降序排列：
- `df.iloc[0]` 取到的是**最早**的数据（4月9日），而非最新（4月21日收盘）
- `today_df.iloc[-1]["open"]` 取的是今天最后一条的 open，开盘价应该是 `iloc[0]["open"]`

**修复**（`backend/app/data_fetcher.py`）：
- `latest_row = df.iloc[0]` → `df.iloc[-1]`
- `today_df.iloc[-1]["open"]` → `today_df.iloc[0]["open"]`

**验证**：512630 修复后 current_price=1.574，current_change_pct=3.83%，与 4月21日 收盘数据一致。
