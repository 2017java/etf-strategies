## 🎯 核心工作流程

```
接收需求 → 方案头脑风暴 → PRD 撰写 → 计划拆分 → A/B 实验 → 数据埋点 → 新用户激活 → 成果概览
```

### 各阶段功能

| 阶段 | 子技能 | 输入 | 输出 |
|------|--------|------|------|
| 1. 需求探索 | brainstorming | 模糊业务需求 | 清晰的产品方案、架构图、功能边界 |
| 2. 需求设计 | write-a-prd | 明确的产品方案 | 专业 PRD 文档、用户故事、验收标准 |
| 3. 计划拆分 | writing-plans | PRD 文档 | 可执行的实施计划、任务清单、里程碑 |
| 4. 实验设计 | ab-test-setup | 待验证功能 | 实验方案、样本量计算、指标体系 |
| 5. 数据准备 | analytics-tracking | 功能特性 | 埋点方案、GTM 配置、数据分析计划 |
| 6. 用户激活 | onboarding-cro | 新功能/产品 | 激活策略、触达方案、A/B 实验计划 |

## 🎨 UI/UX 设计特点

### 设计原则
1. **渐进式信息收集** - 避免一次性要求用户提供所有信息
2. **主动式引导** - 主动提示下一步操作
3. **可视化反馈** - 实时显示工作流进度和状态
4. **清晰的阶段划分** - 每个阶段有明确的开始和结束
5. **响应式设计** - 支持桌面、平板和移动设备

### 配色方案
```css
--primary-color: #2196F3;    /* 产品蓝 - 专业、可信赖 */
--secondary-color: #4CAF50;  /* 成功绿 - 进度、完成 */
--warning-color: #FF9800;     /* 警告橙 - 需要注意 */
--danger-color: #F44336;      /* 错误红 - 需要修复 */
```

## 🚀 如何使用

### 运行演示应用

```bash
# 进入项目目录
cd product-manager-workflow-demo

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 访问应用
# 打开浏览器访问 http://localhost:3000
```

### 在 Coze 平台部署

1. 访问 Coze 官网并注册账号
2. 创建新的应用和智能体
3. 使用提供的 agent-config.json 配置智能体
4. 部署各个技能并配置关联关系
5. 测试完整的工作流
6. 发布到 Coze 应用商店
