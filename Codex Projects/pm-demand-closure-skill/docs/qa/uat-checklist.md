# UAT Checklist (PM Demand Closure)

## 功能链路

- [ ] 模糊输入可以产出结构化 `requirement_brief`
- [ ] 可生成五类核心产物（澄清纪要/PRD/Plan/AB/埋点）
- [ ] 需求含“激活/首登/漏斗”时会触发可选阶段建议

## 质量要求

- [ ] PRD 字段完整率 >= 95%
- [ ] Plan 包含里程碑、任务、依赖、风险
- [ ] AB 卡包含假设、主指标、样本量、停启规则
- [ ] 埋点表包含事件名、属性、触发时机、校验方式

## 规范检查

- [ ] 输出包含 UI/UX 合规检查结果
- [ ] 覆盖 Nielsen、ISO 9241、WCAG 2.2 AA 关键项

## 演示验收

- [ ] 10 分钟内可跑完一次端到端 Demo
- [ ] 交付包可直接用于业务评审
