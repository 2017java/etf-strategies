# UI/UX 合规检查清单（MVP）

## A. 可用性（Nielsen 10）

- [ ] 系统状态可见：关键操作有即时反馈
- [ ] 贴近真实世界：业务术语与页面文案一致
- [ ] 用户可控：支持返回/撤销
- [ ] 一致性：同类操作样式与行为一致
- [ ] 错误预防：关键表单前置校验
- [ ] 识别而非回忆：提供选项与提示，不让用户强记
- [ ] 灵活高效：支持常用快捷流程
- [ ] 极简设计：减少无关信息干扰
- [ ] 错误恢复：报错信息可理解且可执行
- [ ] 帮助文档：高风险环节有说明入口

## B. 交互原则（ISO 9241-110/210）

- [ ] 任务适配：流程围绕业务目标展开
- [ ] 自描述性：页面模块和字段命名清晰
- [ ] 符合预期：行为与用户心理模型一致
- [ ] 可学习性：首次使用可在 5 分钟内上手
- [ ] 容错性：允许纠正输入与重复提交保护

## C. 可访问性（WCAG 2.2 AA）

- [ ] 文本对比度满足最小要求（普通文本建议至少 4.5:1）
- [ ] 交互元素支持键盘访问
- [ ] 有语义化标题与结构标签
- [ ] 表单输入具备关联标签与错误说明
- [ ] 仅颜色不作为唯一信息传达方式

## D. 表单与流程体验

- [ ] 必填项标识清晰
- [ ] 提交前后状态明确（加载/成功/失败）
- [ ] 空态说明包含下一步引导
- [ ] 异常态有降级方案
- [ ] 敏感动作有二次确认
