## 一、产品目标

Inspo 的三组正交目标，分别对应体验、价值和增长：

| # | 目标 | 衡量指标 | 目标值（MVP 3个月） |
|---|------|----------|---------------------|
| G1 | **极速捕获** — 让用户在任何场景下3秒内开始记录灵感 | 记录启动耗时 P50 | <1.5秒（PWA热启动） |
| G2 | **AI增强闭环** — 灵感记下后自动产生可行动产出 | 灵感→AI处理完成率 | ≥90%（含标签/标题/摘要） |
| G3 | **从灵感到执行** — 用户将灵感转化为待办或方案 | 灵感→待办/方案转化率 | ≥25%（活跃用户中） |

三组目标的关系：G1 是入口（没有记录就没有一切），G2 制造"记了就有价值"的正循环，G3 是付费转化的核心驱动力。V1.0 即可验证 G3——手动待办转化功能已纳入 P0，用户记录灵感→AI 自动标签→手动转为待办，形成最小行动闭环。

## 二、用户故事

### US1：产品经理的灵感闪念
> "开会时突然想到一个功能点子，拿出手机2秒内开始打字，写完就走，不用填标题、不用选分类。半小时后再打开，AI 已经帮我打好了 #产品 标签，还生成了一个标题。如果标签不对，我可以在详情页直接点编辑改掉。"

**场景**：会议间隙，单手操作，10-30秒完成一次记录。AI标签可手动修正（P0-12）。

### US2：独立开发者的周日晚间整理
> "一周积累了20多条零散想法。周日晚8点收到推送，打开App看到本周积累了丰富的灵感。进入灵感列表，把最有价值的5条灵感逐条转为待办——设置优先级和截止日期。虽然方案生成要等V1.1，但现在就能把想法变成行动清单。"

**场景**：深度使用，灵感回顾→手动待办转化全链路。"生成方案"是P1功能，但"转为待办"P0就有。

### US3：免费用户探索付费
> "这个月15次免费AI额度用了14次。我在灵感详情页点开AI对话，和AI就一条灵感聊了一轮——它帮我分析了可行性，还推荐了2个竞品。对话中AI建议的行动项，我直接一键转为待办。这种深度体验让我决定升级 PRO。"

**场景**：免费额度试用（15次/月）→ AI对话体验价值感知 + 对话中一键转待办 → 付费转化。

### US4：设计师的视觉灵感
> "看到一张好的交互设计，拍照记录，写了一句描述。AI 自动打了 #UX 标签。后来我回顾时，可以按标签快速筛选所有 UX 相关灵感。"

**场景**：拍照记录→手动写描述→AI标签→按需检索。（图片AI理解MVP不做）

### US5：多设备接力记录
> "地铁上用手机语音记录了一段想法（60秒内免费），到公司后在电脑上打开 PWA 看到转写结果和 AI 标签都已完成。继续编辑完善。"

**场景**：手机语音输入→云端ASR转写→PWA跨端同步。

### US6：清理低价值灵感
> "某天翻到一条没什么价值的灵感——当时随手记的，现在看起来没什么用。左滑删除，干净利落，不占额度也不占列表空间。"

**场景**：灵感列表管理→删除低价值内容→保持灵感库整洁。

## 三、用户研究洞察

### 3.1 目标用户画像

| 维度 | 产品经理 | 独立开发者 |
|------|---------|-----------|
| 日均灵感数 | 8-15条 | 3-8条 |
| 记录场景 | 会议间隙、通勤、睡前 | 开发中、阅读时、深夜 |
| 主要记录方式 | 文字为主（80%），偶尔语音 | 文字（60%）+ 代码片段（20%）+ 链接（20%） |
| 最大痛点 | 记了找不到、记了不行动 | 想法散落各处、无法串联成方案 |
| 付费意愿 | 中等（公司报销可能性高） | 高（自己说了算，ROI敏感） |

### 3.2 关键UX原则

- **拇指区优先**：核心操作区占据屏幕底部70%，适配单手操作
- **零摩擦记录**：打开即记，无需任何前置操作（选分类/填标题/打标签）
- **渐进式AI呈现**：AI标签和摘要不是记录的前提，而是记录的结果——后台异步处理，用户无感知
- **30秒首次闭环**：新用户从打开到看到AI处理结果 ≤30秒
- **火焰 Streak 激励**：7级火焰视觉（灵感日历连续记录天数），利用损失厌恶驱动留存
- **灵感冻结卡**：连续3天未记录可触发，赠送1次方案生成额度，拉回沉睡用户

### 3.3 免费版AI额度分配策略

免费版每月 **15次** AI额度（记录一条灵感→生成标签/标题/摘要 = 消耗1次）。成本模型：

| 模型 | 单次标签成本 | 15次/月/人 | 千人月成本 | 万人月成本 |
|------|------------|-----------|-----------|-----------|
| DeepSeek V4 Flash（主） | ¥0.00072 | ¥0.011 | **¥11** | **¥108** |
| 豆包Lite（备选） | ¥0.0002 | ¥0.003 | ¥3 | ¥30 |

15次覆盖日均0.5次的基础标签需求。用完触发AI额度降级，让用户感知"AI受限"的落差，驱动付费转化。无需"5-2-2-1"紧张分配策略。

**用完后的四级平滑降级**（禁止粗暴弹窗）：
1. **智能排队**：剩余额度>0但<正常处理阈值时，AI处理进入低优先级队列（仍处理但延迟<5分钟）
2. **预览诱惑**：额度=0时，展示AI标签/标题的前几个字，完整内容模糊+"升级PRO解锁"
3. **手动模式**：额度=0持续3天，引导手动标签
4. **惊喜回馈**：连续5天未活跃+额度=0+注册>14天→随机赠送1-3次额度（每月≤1次）

### 5.1 AI API选型及成本模型

**AI能力提供商**：**火山方舟 CodingPlan API**（用户提供的URL+API Key），支持多模型切换（DeepSeek V4 Flash、豆包系列、Kimi、GLM等）。

**标签生成场景（场景A）— DeepSeek V4 Flash**：

| 参数 | 数值 |
|------|------|
| 模型 | DeepSeek V4 Flash（通过CodingPlan调用） |
| 输入（系统prompt 200 + 用户125字×1.8） | ~425 tokens |
| 输出（标签≤5个，均值3个，~135 tokens） | ~135 tokens |
| 输入价格 | $0.14 / 1M tokens |
| 输出价格 | $0.28 / 1M tokens |
| **单次标签成本** | **≈ $0.0001 ≈ ¥0.00072** |

> ⚠️ 通过火山引擎 CodingPlan 调用可能有渠道加价，预估加价50-100%后单次成本仍在¥0.001-0.0015区间。千人月成本<¥45。

| 场景 | 模型建议 | 消耗倍数 | 单次成本（估算） | 说明 |

## 六、需求池

### P0 — MVP 核心（V1.0，8周）

| ID | 需求 | 验收标准（Given/When/Then） | 优先级 |
|----|------|---------------------------|--------|
| P0-01 | 文字极速记录 | Given 用户打开PWA首页 When 页面加载完成 Then 输入框自动聚焦，用户可直接打字，发送后卡片即时出现在下方feed。从打开到首字输入<1.5秒 | P0 |
| P0-02 | 语音记录 | Given 用户在首页 When 长按麦克风按钮说话 Then 录音保存，异步转写为文字填入卡片，同时保留原始音频可播放。免费版限60秒/次（倒数10秒软toast+振动提醒→到点掐断），PRO版5分钟/次（倒数30秒软toast+振动→到点掐断）。**取消手势**：长按录音时手指上滑到取消区域（屏幕上方1/3），UI变为红色取消区域提示+振动，松手=取消录音不保存。**转写失败**：火山ASR转写失败时卡片显示"🎤语音记录(MM:SS)—转写失败 [重试]"，原始音频保留可播放。**音频编码**：WebM Opus格式，目标压缩至~200KB/分钟 | P0 |
| P0-03 | 拍照记录 | Given 用户在首页 When 点击相机按钮拍照 Then 照片回填为卡片配图，用户手动输入描述文字。MVP不做AI图片理解 | P0 |
| P0-04 | 离线同步引擎 | **同步策略**：Last-Write-Wins(LWW)，基于updated_at时间戳，后写入者覆盖先写入者。**同步状态机**：local→syncing→synced（成功）或conflict（冲突可查看）。**存储配额**：IndexedDB总上限200MB，超出时优先清理已同步的音频文件，其次清理已同步的图片，再次提示用户清理。**网络切换保护**：WiFi↔4G切换时同步队列幂等重试，基于client_id去重，不丢数据不重复。**同步失败重试**：指数退避策略(1s→2s→4s→max 60s)。**Service Worker缓存策略**：静态资源Cache First，API Network First，灵感数据不缓存 | P0 |
| P0-05 | AI自动标签+标题+摘要 | Given 用户保存了一条灵感 When AI异步处理完成 Then 卡片自动填充标签（**最多5个**，通常1-3个）、10字内标题、20字内摘要。标签置信度<0.4时标记为"待分类"。1条灵感生成标签=消耗1次AI额度。**AI处理状态机**：AI处理超时30秒→自动重试1次→仍失败→标记"AI处理失败[重试]"。CodingPlan返回5xx→重试最多3次→失败标记。JSON解析失败→fallback从文本中提取标签→仍失败标记"待分类"。**前端展示**：卡片初始显示骨架屏+微文案"AI正在理解..."，30秒后无结果显示"处理中，稍后自动完成"。**额度检查**：标签生成前进行额度检查——有额度且未处理过(inspiration_id去重)→扣减1次→调用AI→成功回填/失败退额度 | P0 |
| P0-06 | PWA安装与离线 | Given 用户在浏览器中访问Inspo When 符合PWA安装条件 Then 显示"添加到主屏幕"引导。安装后Service Worker缓存核心资源，热启动<1秒 | P0 |
| P0-07 | 用户注册登录 | Given 未登录用户 When 完成首次灵感记录并看到AI结果后 Then 引导注册。**浏览器环境自适应**：微信内置浏览器→微信OAuth一键授权；系统浏览器(Safari/Chrome)→手机号+验证码为主入口，微信扫码登录为备选。Supabase Auth免费用5万用户/月。首次注册后本地IndexedDB中的离线记录自动关联到账户 | P0 |
| P0-08 | 灵感列表与筛选 | Given 用户有多条灵感 When 进入灵感列表 Then 可按时间排序，按标签筛选（顶部横向滚动标签栏），支持关键词搜索 | P0 |
| P0-09 | AI额度系统 | Given 免费用户本月AI额度已用完15次 When 再次触发AI处理 Then 按四级降级策略处理。**四级降级触发条件**：第1级"智能排队"——剩余额度>0但<正常处理阈值时，AI处理进入低优先级队列（仍处理但延迟<5分钟）；第2级"预览诱惑"——额度=0时，展示AI标签/标题前几个字，完整内容模糊+"升级PRO解锁"；第3级"手动模式"——额度=0持续3天，引导手动标签；第4级"惊喜回馈"——连续5天未活跃+额度=0+注册>14天→随机赠送1-3次额度（每月≤1次）。**额度扣减**：数据库层原子操作(PostgreSQL UPDATE...WHERE remaining>0 RETURNING)，不在应用层计数。**额度重置**：每月1日0:00通过Supabase pg_cron重置。显示剩余额度（首页头像旁）。"消耗1次"定义：1条灵感记录→AI生成标签/标题/摘要 = 消耗1次 | P0 |
| P0-10 | 灵感详情页AI对话 | Given 用户查看一条灵感详情 When 点击"AI对话"按钮 Then 进入对话框模式，可就此灵感与AI深度对话（可行性分析、竞品推荐、功能拆解等）。**"1轮对话"定义**：1轮=用户发送1条消息+AI回复1条，每轮消耗1次额度。单条消息不超过2000字符。**额度耗尽处理**：对话中途额度耗尽→AI返回"本次额度已用完，升级PRO继续对话"+已生成内容不丢失。**对话中待办转化**：AI建议的行动项可一键转为待办（复用P0-11的待办创建面板） | P0 |
| P0-11 | 灵感→待办手动转化 | Given 用户在灵感详情页 When 点击"转为待办"按钮 Then 弹出待办编辑面板，预填标题=AI标题、详情=灵感原文，用户可调优先级(高/中/低)和截止日期后保存。待办自动关联来源灵感。待办列表：按"待完成/已完成"分组，勾选即完成，支持按优先级排序。这是最小闭环——不做AI批量提取（P1）、不做日历（P1）、不做方案生成（P1） | P0 |
| P0-12 | 标签手动修正 | Given 用户查看一条有AI标签的灵感 When 点击标签旁的编辑图标 Then 弹出标签编辑面板，可删除/添加标签（最多5个），从7个预设一级标签中选择。手动修正的标签覆盖AI标签，后续不再自动修改。用户未修正的标签保持AI标签不变 | P0 |
| P0-13 | 灵感删除与编辑 | Given 用户查看一条灵感 When 点击"更多"(⋮)按钮 Then 弹出菜单：编辑内容/删除灵感。**删除灵感**：确认后从列表移除+IndexedDB标记删除+同步删除Supabase记录。**编辑灵感**：进入编辑模式可修改文字内容，保存后更新 | P0 |

## 八、Non-goals（明确不做）

以下功能**明确排除**在 MVP 及近期路线图中：

| 不做 | 原因 |
|------|------|
| ❌ Demo/原型生成 | 产品定位是"灵感到方案"而非"方案到Demo"。跨到代码生成会模糊核心价值且大幅增加AI成本 |
| ❌ 图片AI理解（OCR/描述生成） | MVP仅保存照片+用户手动描述。多模态模型成本高且当前能力不足以准确理解灵感类图片，V1.1重新评估 |
| ❌ 原生App（iOS/Android） | MVP验证用PWA，验证PMF后再考虑原生。PWA已覆盖离线、推送、安装到桌面等核心能力 |
| ❌ 多人协作/团队版 | 个人工具先行，协作是规模化后的方向 |
| ❌ 公开灵感社区/广场 | 灵感是私密的，社区功能与产品气质不符 |
| ❌ 自定义AI Prompt模板 | 初期固定Prompt确保输出质量一致，V2可开放 |
| ❌ 第三方集成（飞书/Notion/Slack） | MVP聚焦核心闭环，集成在P2规划 |
| ❌ AI生成代码/设计稿 | 超出"方案"范畴，属于另一个产品 |
| ❌ Web端独立域名（桌面优先） | MVP以移动端H5+PWA为主，桌面端仅保证基础可用 |
| ❌ 用户自定义标签体系 | MVP使用固定7大类+子标签预设，不让用户从零建标签体系 |
| ❌ 灵感公开分享/社交功能 | 灵感私密性第一，分享仅限方案导出 |
| ❌ Redis | MVP用Supabase内置能力即可满足，不过度工程 |
| ❌ CRDT同步 | MVP用Last-Write-Wins(LWW)方案，简化实现 |
| ❌ 自建数据分析后台 | 用Supabase Dashboard + Umami自部署满足报表需求，暂不自建 |

## 十二、安全性设计

### 12.1 Supabase RLS策略
- 所有用户数据表(user_inspirations, tags, todos, quotas)必须启用Row Level Security
- 策略：用户只能读写自己的数据(auth.uid() = user_id)
- 建表时同步创建RLS策略，不在应用层做权限校验

### 12.2 API Key安全
- 火山方舟CodingPlan API Key仅存在于Supabase Edge Function Secret环境变量中
- 客户端永远不接触API Key
- 所有AI调用通过Edge Function代理

### 12.3 数据隐私
- 传输加密：全链路HTTPS
- AI请求中去掉用户身份标识（仅传灵感内容+历史上下文）
- V1.0上线前完成隐私政策页面
